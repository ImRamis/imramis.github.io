PoC: Fullscreen API Address Bar Spoofing

Target: Chrome 145 | Tests fullscreen mode to overlay fake browser UI over real address bar

Related: CVE-2023-2930, CVE-2022-3046, CVE-2021-30625 (fullscreen spoofing)

Test 1: Basic fullscreen with fake Chrome UI

Enters fullscreen mode and overlays a pixel-perfect fake Chrome address bar showing "accounts.google.com".

Chrome shows "Press Esc to exit full screen" briefly — this test waits for that to disappear.






Test 2: Fullscreen with delayed UI spoof



  
Enters fullscreen, waits for Chrome's notification to disappear (~3s), then shows fake UI.

  
  





Test 3: Fullscreen escape suppression



  
Tests if Esc key can be intercepted/suppressed while in fullscreen to prevent exit.

  
  





Test 4: Fullscreen via Pointer Lock combo



  
Combines fullscreen with Pointer Lock to prevent mouse from reaching real UI elements.

  
  





Test 5: Fullscreen from iframe



  
Tests if an iframe can enter fullscreen and the parent controls what fake UI is displayed.

  
  

  




Test 6: Keyboard event interception in fullscreen



  
While in fullscreen, intercepts keyboard events to capture credentials typed into fake login form.

  
  





Test 7: Fullscreen notification style matching



  
Studies Chrome's fullscreen exit notification and creates a fake replacement after it fades.
