PoC: Navigation Race Conditions

Target: Chrome 145 | Tests race conditions in navigation that could confuse security UI

Related: CVE-2023-2721, CVE-2022-3373, CVE-2021-30598 (navigation races)

Test 1: Rapid location.href toggling

Rapidly sets location.href back and forth between two URLs. Tests if URL bar shows stale state.






Test 2: window.stop() during navigation commit



  
Starts navigation then calls window.stop() at various timings. Tests if URL bar shows the cancelled URL.

  
  

  




Test 3: back()/forward() rapid cycling



  
Rapidly alternates between history.back() and history.forward() to desync URL bar state.

  
  
  





Test 4: iframe src + location.replace race



  
Sets iframe src and immediately calls contentWindow.location.replace. Tests which URL wins and what's displayed.

  
  

  




Test 5: form.submit() + location change race



  
Submits a form while simultaneously changing location. Tests navigation priority and URL bar state.

  
  

  

    
  

  




Test 6: setTimeout(0) navigation cascade



  
Queues multiple navigations via setTimeout(0). Tests microtask/macrotask ordering impact on URL bar.

  
  





Test 7: window.open + immediate navigation replacement



  
Opens window to URL A then immediately tries to navigate it to URL B. Tests URL bar state.

  
  





Test 8: beforeunload + navigation race



  
Uses beforeunload event handler to trigger additional navigations during page unload.

  
  





Test 9: requestAnimationFrame navigation timing



  
Uses rAF to time navigation attempts at the boundary of frame rendering, when URL bar may be mid-update.
