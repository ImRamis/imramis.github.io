PoC: Blob URL Origin Spoofing

Target: Chrome 145 on Linux | Tests blob: URL origin display and navigation confusion

Related: CVE-2023-2312, CVE-2022-3889 (blob URL origin issues)

Test 1: Blob URL with spoofed origin display

Creates a blob URL whose content mimics a legitimate site. Tests if the blob: origin in the address bar is misleading.






Test 2: Blob URL navigation then history manipulation



  
Opens blob URL, then uses history.replaceState to change displayed URL.

  
  





Test 3: Blob URL with meta-refresh redirect



  
Creates blob URL with meta-refresh that redirects. Tests if URL bar updates correctly during transition.

  
  





Test 4: Revoking blob URL during navigation



  
Creates blob URL, starts navigation, then revokes it mid-flight. Tests error state display.

  
  





Test 5: Blob URL iframe with srcdoc fallback



  
Tests blob URL in iframe with srcdoc, checking origin inheritance chains.

  
  

  




Test 6: Nested blob URLs (blob within blob)



  
Creates a blob URL that itself creates another blob URL. Tests origin chain and address bar behavior.

  
  





Test 7: Blob URL + window.open + opener access



  
Tests if blob URL opened via window.open retains opener reference and can manipulate opener's location.

  
  





Test 8: Blob URL with Service Worker interception



  
Tests if a Service Worker can intercept navigation to a blob URL and serve different content.
