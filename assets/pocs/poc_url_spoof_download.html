PoC: Download Dialog & Content-Disposition Spoofing

Target: Chrome 145 | Tests download UI for origin confusion and filename spoofing

Related: CVE-2023-2929, CVE-2021-30625 (download UI spoofing)

Test 1: Blob download with misleading filename

Creates a blob download with a filename that looks like it's from a trusted source.






Test 2: Download with RLO in filename



  
Uses Right-to-Left Override to make filename extension appear different (e.g., exe→txt).

  
  





Test 3: Download triggered during navigation



  
Triggers download while page is navigating, testing if download dialog shows correct origin.

  
  





Test 4: Multiple simultaneous downloads



  
Triggers many downloads at once to overwhelm the download UI and potentially confuse origin display.

  
  
  





Test 5: Data URL download with Content-Disposition simulation



  
Tests data: URL downloads and how Chrome handles the filename/origin display.

  
  





Test 6: Download via iframe sandbox



  
Triggers download from a sandboxed iframe. Tests if origin in download dialog matches iframe or parent.

  
  

  




Test 7: Download with very long filename



  
Tests if extremely long filenames can push the real extension off-screen in download bar.

  
  





Test 8: Navigation + download + history race



  
Combines pushState URL change with download trigger to confuse which origin initiated the download.
